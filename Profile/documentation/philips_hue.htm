Summary:

This profile is created to control the Philips Hue lighting device. This profile follows our data table format for lighting controllers, in that we will be able to auto-populate the UI with most of the Hue’s native functionality. Through the data table we are able to control the Brightness, Hue, and Saturation for each individual bulb that is connected. Out of the three control modes that are available on this device (Hue/Saturation, XY Color, and Color Temperature) Savant enabled control using the Hue/Saturation mode. The other control modes are available in the custom section of the profile, but the data table will only provide access to the Hue/Saturation mode. Savant determined that the Hue/Saturation mode will give the highest range of color and functionality, and will integrate the easiest with Savant. Because of this, the Savant system may not accurately reflect the device’s current color if controlled by any other besides Savant. Therefore it is recommended that Savant be the primary controller of this device once installed.

Setting Up in Blueprint:

1)   Search for the Phillips Hue profile and place in the system. 

2)   Connect the Ethernet connection on the Hue component.

3)   Open Inspector for this Ethernet connection and enter the IP address of the Hue bridge in the Host address field.

4)   Open in Inspector for the Hue component and Click on the Data Table button.

5)   Add all of the entities you wish for each bulb. The specific entities are described below. (It is very possible you will have multiple entities per bulb. Figure 1 in the Appendix shows an example data table setup for three bulbs.) The addresses for each bulb can be found in the color picker screen of the Hue app. (See Figure 3 below.)

6)   Make sure Lighting auto population checkbox is selected in the Preferences panel, UI Defaults Tab.

7)   Generate Services, Sync with Services as you normally would.

8)   If you have configured your data table as shown in Figure 1, it is recommended that you add a text field to each page to denote which bulb that page is currently controlling.

9)   Sync screens and save and upload the config before continuing to the Connecting to the Device section of this guide

 

Setting Up RGB Sliders:

If you would like to use a Red, Green, and Blue slider instead of what is given by default, follow steps 1-4 in the “Setting Up in Blueprint” section then

1)   Add three entities for each light you wish to have RGB sliders for.

2)   Select RGB for each entity.

3)   In Address 1, enter the address of the bulb you wish to control, and in Address 2 you need to put 0, 1, or 2, depending if you want this to control Red, Green, or Blue. (Red=0, Green=1, Blue=2)

4)   Repeat these steps 1-3 for each light you would like to have RGB sliders for.

5)   Generate services, sync screens, and save and upload the config before continuing to the Connecting to the Device section of this guide

Setting Up on the Device:

IP:

1)   Plug in the Hue bridge, connect to the network, and make sure it gets assigned an IP address on the same subnet as the Savant system. We highly recommend either giving the bridge a static IP address, or creating a DHCP reservation for the Hue bridge in your router

 

2)   Use the Hue app to setup and assign any bulbs that you have before proceeding with the Savant setup

 

 

Connecting to the Device:

IP:

1)   Connect the Hue bridge and make sure that is assigned an IP address.

2)   Once the Hue bridge is on the network, you will need to sync it to the Savant system. To do this:

a.     First press the link button on the top of the Hue bridge.

b.     Next, within a short amount of time after pressing the link button, (less than a minute,) you need to send the RegisterDevice custom command from the profile. You can send this as an Automator action from the host. .

FAQ/Common Mistakes:

1)   I am not able to control the Hue from my Savant interface. 

a.     Make sure that the Hue bridge is on the network and is accessible through the Hue app.

b.     Make sure the IP address of the Hue is on the control wire in Blueprint.

 

2)   What to the different entities do in the data table? 

a.     Brightness: This is a slider that controls the brightness of the addressed bulb. This goes from 0 (very dim) to 100 (very bright). Note that a brightness of 0 will not turn the bulb off completely.

b.     Hue: This is a slider that changes the actual Hue (or color) of the bulb and represents the different colors of the spectrum that the Hue bulbs support.

c.      Saturation: This slider does not affect the actual color of the bulb, but will change the bulb from washed out (0) to vibrant color (100)

d.     On/Off: This is a toggle button that will turn on and off the current bulb.

e.     Flash: This push button will momentarily flash the current bulb. This may not be needed in your implementation, but is helpful in finding the current bulb if the pages are not labeled.

f.      RGB: This will create a red, green, or blue slider depending on what you have set in the Address 2 field.

 

3)   How do I find the addresses of my Hue bulbs? The addresses from each bulb can be seen in the color picker screen in the Hue app. The addresses are the numbers that you drag around the screen to pick the color of the bulb. You can see addresses 1, 2, and 3 available in Figure 3.

 

4)   How do I control more than one bulb at once? At this time the Hue API does not fully support group commands. It is expected in a later release of their API, but for now you will need to create custom workflows that can control more than one bulb at a time.

 

5)   I changed a light from the Hue app, but the change doesn’t show on the Savant interface?   This is caused when the Hue app (or any other controller besides the Savant app) change the bulb to something other than Hue/Saturation mode. Since we are only controlling this device in Hue/Saturation mode, any other modes that are used will not reflect on the interface.

 

6)   Can I use XY color mode or Color Temperature mode?  Yes, the commands to use these modes are included as custom actions in the current profile. We chose not to include these in the default UI for this device, but these can be used by creating custom workflows and custom UI elements.

 

 

State Variables/Feedback:

HVAC Service

DT_Saturation_#

Keeps track of Saturation level of the sliders in the data table.

UniqueID: CurrentLightNumber

Type: string

Range: 0-100

 

DT_Hue_#

Keeps track of Hue level of the sliders in the data table.

UniqueID: CurrentLightNumber

Type: string

Range: 0-100

 

DT_Brightness_#

Keeps track of Brightness level of the sliders in the data table.

UniqueID: CurrentLightNumber

Type: string

Range: 0-100

 

isLightOn_#

Keeps track of the on/off status of the current light.

UniqueID: CurrentLightNumber

Type: boolean

 

Custom Variables

LightPowerStatus_#

String to display the on/off status of the current light

UniqueID: CurrentLightNumber

Type: string

Range: “OFF” or “ON”

 

Brightness_#

Keeps track of the brightness of the current light.

UniqueID: CurrentLightNumber

Type: string

Range: 0-254

 

Hue_#

Keeps track of the hue of the current light.

UniqueID: CurrentLightNumber

Type: string

Range: 0-65535

 

Saturation_#

Keeps track of the saturation of the current light.

UniqueID: CurrentLightNumber

Type: string

Range: 0-255

 

XValue_#

Keeps track of the X color value of the current light.

UniqueID: CurrentLightNumber

Type: string

Range: 0.0000 – 1.0000

 

YValue_#

Keeps track of the Y color value of the current light.

UniqueID: CurrentLightNumber

Type: string

Range: 0.0000 – 1.0000

 

ColorTemp_#

Keeps track of the color temperature value of the current light.

UniqueID: CurrentLightNumber

Type: string

Range: 100-500

 

Alert_#

String that displays any alerts received from the light.

UniqueID: CurrentLightNumber

Type: string

 

Effect_#

String that displays the current effect of the light.

UniqueID: CurrentLightNumber

Type: string

 

ColorMode_#

String that displays the current color mode of the light.

UniqueID: CurrentLightNumber

Type: string

 

isLightReachable_#

Boolean state that tells whether or not the current light is reachable by the Hue bridge.

UniqueID: CurrentLightNumber

Type: boolean

 

 

Custom Work Needed:

No Custom work is REQUIRED.  However if you followed the data table example in this guide, you may want to add the current light name to each page that has sliders on it. Also you will need custom work if you wish to use the XY or Color Temperature modes for the bulbs.
